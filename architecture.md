# Архитектура рекомендательной системы

## 1. Базовый принцип

Система использует чистую коллаборативную фильтрацию.  
Единственный источник сигналов — реакция пользователя на фильмы:

- оценка  
- досмотр  
- пересмотр  
- лайк  

Метаданные (жанр, страна, возраст и т.д.) не используются в базовой модели.

---

## 2. Структура данных

Формируется матрица взаимодействий:

Фильм × Пользователь → Реакция

- строки — идентификаторы фильмов  
- столбцы — идентификаторы пользователей  
- ячейка — оценка или поведенческий сигнал  

Каждый пользователь представлен своим столбцом оценок.

---

## 3. Определение сходства

Сходство между пользователями вычисляется на основе пересечения оценённых фильмов.

Используется метрика сходства (например, косинусное сходство или корреляция).

Если коэффициент сходства ≥ 0.9, пользователи считаются близкими по вкусам.

---

## 4. Кластеризация пользователей

Пользователи группируются в кластеры по высокому уровню сходства.

Каждый кластер представляет собой плотную группу пользователей со схожими предпочтениями.

### Алгоритм формирования кластеров:

1. Выбирается пользователь.
2. Находятся пользователи с коэффициентом сходства ≥ порога.
3. Формируется кластер.
4. Пользователи исключаются из дальнейшей кластеризации.
5. Процесс повторяется для оставшихся пользователей.

---

## 5. Генерация рекомендаций

Рекомендации формируются внутри кластера:

- пользователю предлагаются фильмы, которые получили высокие оценки от других участников его кластера;
- фильмы, уже оценённые пользователем, исключаются;
- результаты ранжируются по средней или взвешенной оценке внутри кластера.

---

## 6. Обработка новых фильмов

Для нового фильма применяется этап "разогрева":

1. Фильм показывается ограниченному числу пользователей внутри подходящих кластеров.
2. Собираются первичные оценки.
3. После накопления достаточного количества реакций фильм становится частью стандартного процесса рекомендаций.

---

## 7. Масштабирование

Архитектура допускает:

- увеличение числа пользователей;
- увеличение числа фильмов;
- оптимизацию вычислений при росте нагрузки.

Базовая модель остаётся неизменной: рекомендации строятся исключительно на основе сходства реакций пользователей.

---

## Итог

Система представляет собой реакционно-ориентированную кластерную модель рекомендаций, где персонализация достигается через группировку пользователей с близкими поведенческими паттернами.
